# text_nav_bridge

Text-command-driven robot navigation for ROS2. Reads text landmarks generated by TextMap (Phase 1) and converts user text commands into Nav2 `NavigateToPose` actions.

## Features

- Text similarity matching using Levenshtein distance
- Automatic closest landmark selection when multiple matches exist
- Configurable approach distance to stop before the landmark
- RViz goal marker visualization
- Navigation status feedback via topic

## Components

- **text_nav_bridge_node**: C++ node that subscribes to text commands, looks up matching landmarks, and sends navigation goals to Nav2.

## Inputs and Outputs

### Inputs
- Command: `/text_nav/command` (std_msgs/String) — text to navigate to
- TF: `map` → `camera_link` (via rtabmap localization)
- Landmark file: `landmarks.yaml` generated by TextMap in Phase 1

### Outputs
- `/text_nav/status` (std_msgs/String) — navigation status updates
- `/text_nav/goal_marker` (visualization_msgs/Marker) — goal visualization in RViz

## Dependencies

- ROS 2 Humble
- [TextMap](https://github.com/hyunsehyun/TextMap) — text landmark SLAM (provides shared `text_similarity.hpp`)
- [rtabmap_ros](https://github.com/hyunsehyun/rtabmap_ros) — Visual SLAM and localization
- [navigation2](https://github.com/hyunsehyun/navigation2) — Nav2 navigation stack
- [NavOCR](https://github.com/hyunsehyun/NavOCR) — sign detection and OCR (used in Phase 1)
- yaml-cpp, tf2, rclcpp_action

## Build

```bash
cd ~/ros2_ws
colcon build --packages-select text_nav_bridge
source install/setup.bash
```

## Usage

### Phase 1: Mapping

Record environment data and build the map with text landmarks.

**1. Record rosbag (real camera)**
```bash
ros2 launch rtabmap_ros realsense_infra_for_record.launch.py
# In a separate terminal
ros2 bag record -a -o my_session
```

**2. Run mapping (rosbag playback + SLAM + OCR)**
```bash
ros2 launch textmap textmap.launch.py rosbag_path:=/path/to/my_session landmark_save_path:=/path/to/landmarks.yaml
```

This launches rosbag playback, static TF, IMU filter, RGBD odometry, RTAB-Map SLAM, NavOCR, and TextMap all at once.

After Phase 1, two outputs are produced:
- **RTAB-Map DB** (`~/.ros/rtabmap.db`) — 3D map of the environment
- **landmarks.yaml** — text landmark list with 3D coordinates

```yaml
landmarks:
  - id: 1
    text: "restroom"
    position: {x: 3.21, y: -1.05, z: 0.82}
    confidence: 0.92
    observation_count: 5
  - id: 2
    text: "exit"
    position: {x: 7.43, y: 2.11, z: 0.95}
    confidence: 0.87
    observation_count: 3
```

### Phase 2: Navigation

Navigate using text commands in the mapped environment.

**1. Launch navigation stack**
```bash
ros2 launch text_nav_bridge text_nav.launch.py landmark_file:=/path/to/landmarks.yaml
```

This launches RTAB-Map localization, Nav2, and text_nav_bridge.

**2. Send a text command**
```bash
ros2 topic pub --once /text_nav/command std_msgs/msg/String "data: 'restroom'"
```

text_nav_bridge will:
1. Find the best matching landmark from `landmarks.yaml`
2. Compute a goal point `approach_distance` meters in front of the landmark
3. Send a `NavigateToPose` goal to Nav2

**3. Visualize in RViz**
```bash
rviz2 -d $(ros2 pkg prefix text_nav_bridge)/share/text_nav_bridge/rviz/text_nav.rviz --ros-args -p use_sim_time:=true
```

The provided `text_nav.rviz` config includes the following displays:

| Display | Topic | Description |
|---------|-------|-------------|
| Map | `/map` | 2D occupancy grid from RTAB-Map |
| TF | — | Robot position (`map` → `odom` → `camera_link`) |
| Global Plan | `/plan` | Full path from robot to goal (green line) |
| Local Plan | `/local_plan` | Real-time obstacle-avoidance path (yellow line) |
| Local Costmap | `/local_costmap/costmap` | Cost map around the robot reflecting obstacles |
| Nav Goal Marker | `/text_nav/goal_marker` | Arrow marker at the navigation goal |
| Text Landmarks | `/textmap/markers` | Text landmark labels from Phase 1 |
| SLAM Path | `/rtabmap/mapPath` | Historical trajectory from RTAB-Map |

Global Costmap (`/global_costmap/costmap`) and Point Cloud (`/rtabmap/cloud_map`) are also included but disabled by default to reduce rendering load. Enable them in the RViz Displays panel as needed.

**4. Monitor status**
```bash
ros2 topic echo /text_nav/status
```

Example output:
```
data: "NAVIGATING: heading to 'restroom'"
data: "NAVIGATING: distance_remaining=2.31m"
data: "NAVIGATING: distance_remaining=0.85m"
data: "SUCCESS: Navigation completed"
```

## Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `landmark_file` | (required) | Path to landmarks.yaml |
| `match_threshold` | 0.5 | Text similarity threshold (0~1). Lower = more lenient matching |
| `approach_distance` | 1.0 | Stop this many meters before the landmark |
| `robot_frame` | `camera_link` | Robot base frame |
| `world_frame` | `map` | World coordinate frame |
